# Playwright 页面级录屏与素材抓取 Playbook

更新时间：2026-06-04

## 适用目标

用于 AI 优化商品图推广视频的干净后台素材采集：

- Playwright 自动操作目标页面，并只录制目标 page viewport。
- 后台生成结束后进入换图记录 taskId 页。
- 在确认页批量抓图片链接并下载素材。
- 等待时间后期加速，关键操作点保留正常速度。
- Remotion 使用干净页面录屏和商品素材生成视频母版。
- Jianying Editor 基于 Remotion 母版生成剪映草稿，补口播字幕、花字、BGM、TTS、大字标题。
- 点击确认发布，形成“生成 -> 查看结果 -> 自动发布/上架”的闭环证据。

本流程里的“录屏展示 hover”只服务于视频画面；素材抓取不依赖 hover，必须从页面 DOM 图片链接批量抓取。

## 标准链路

```text
Playwright 自动操作页面
  -> Playwright 录目标页面 viewport
  -> DOM/API 抓原图和生成图
  -> 等待时间加速，操作加速，关键点正常速度
  -> Remotion 使用干净页面录屏 + 商品素材生成干净母版
  -> Jianying Editor 生成剪映包装草稿
  -> 用户在剪映中预览并手动导出
```

## 录制原则

主方案必须是页面级录制，不再录整个屏幕。

- 只录目标页面 viewport，不录地址栏、系统弹窗、保存框、其他窗口。
- 用户切换到其他页面或其他 App，不影响正在录制的目标页面画面。
- 浏览器窗口尺寸固定，推荐按后续视频结构设置为 `1920x1080` 或可裁切的横版 viewport。
- Playwright 控制一个独立 browser context 或持久化 user data dir，避免污染用户当前手动浏览。
- 录制产物先作为干净素材，不直接作为最终成片。

兜底方案：

- ffmpeg/系统录屏只保留为流程证据，不作为可投放视频素材。
- 如果必须用窗口录制，也要单独开独立 Chrome 窗口并固定区域；但仍不如 Playwright page video 稳定。

## 速度处理规则

页面录制保留真实操作顺序，但最终进入 Remotion 前要生成一份节奏版录屏：

- 等待生成阶段：加速到 `8x-20x`，只保留状态变化证据。
- 普通滚动、搜索、列表加载：加速到 `2x-4x`。
- 关键配置点：正常速度，必要时轻微慢放。
- 点击 `立即生成`、进入 `确认生成结果`、点击 `确认并发布`：正常速度，并预留局部放大空间。
- hover 查看大图：只用于录屏展示，不参与素材抓取。

- 店铺/平台选择。

- 主图 1 支持卖点。
- 不带显式 AI 标识。

## 素材抓取方式

优先在确认页执行一次性 DOM 导出：

1. 收集 `document.images`。
2. 保留 `currentSrc/src/alt/naturalWidth/naturalHeight`。
3. 用相邻行文本关联商品 ID。
4. 原始 JSON 保存为 `task_{taskId}_dom_assets_raw.json`。
5. 结构化时按每个商品 11 张图拆分：
   - 第 1 张：原图。
   - 第 2-6 张：1:1 生成图。
   - 第 7-11 张：3:4 生成图。
6. OSS 图片下载时去掉 `x-oss-process=image/resize,w_120/quality,q_85` 参数，优先下载原始尺寸。

## 后续第一幕模板约束

第一幕不再只放单个商品，也不做拼图。

结构：

- 3 组商品按行排列。
- 左侧先缓慢出现 3 张原图，顶部标注 `原图`。
- 右侧区域顶部标注 `生成效果图`。
- 每个商品的生成图逐张入场。
- 生成图出现时先放大到明显大图状态，再缩小贴到对应行的坑位。
- 右侧功能文案始终存在，但只占约 1/3 空间，字体要更大更清晰。
- 核心表达是“优化套图”，不是只优化一张主图。

节奏：

- 第一幕至少 6 秒。
- 生成效果图入场要慢，最后全部出现后定住 1 秒。
- 大图放大效果至少是前版的 2 倍，保证主图感和冲击力。

## 可接受边界

- 测试店铺和测试商品允许点击 `立即生成` 与 `确认并发布`。
- 页面上的 hover 只用于录屏展示，不作为素材采集依赖。
- 如果 Chrome 禁止 AppleScript/DevTools 注入，允许使用地址栏 bookmarklet 导出 DOM 图片链接。
- 如果 bookmarklet 首次粘贴被 Chrome 去掉 `javascript:` 前缀，需要重新设置完整地址栏值再执行。
- 新标准下优先使用 Playwright `page.evaluate` 或接口响应捕获抓素材；bookmarklet 仅作为人工兜底。
