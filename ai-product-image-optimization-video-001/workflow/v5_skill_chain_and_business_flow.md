# AI 商品图优化短视频生产流程 v5 记录

更新时间：2026-06-03

## 已固化的样片细节

- 一个视频只讲一个案例：当前样片只讲童装，不再混入女装。
- 核心卖点聚焦上架主图：一次生成 5 张主图，直接组成一套上架。
- 删除“挑 1-2 张大图对比原图”。
- 删除“有的适合上架，有的适合发笔记”。
- 批量段核心表达：批量选中商品，批量生成全套主图视觉。
- 字幕字号需要比 v4 更大，并放在社媒平台底部 UI 安全区以上。
- 录屏区域需要尽量放大，竖向区域多一些，减少空白。
- 录屏必须露出左侧功能入口和关键操作区。
- 录屏步骤文案统一为：第一步、第二步、第三步。
- 结果展示必须使用童装 5 张效果图，逐张 hover 放大检查。
- 收尾展示批量结果列表和「确认并发布」，但不写外部导购 CTA。

## Skill 串联方式

### 1. Office Hours / 需求压缩

用途：把想法压成可验证的单条样片目标。

输入：

- 推广目标：推广 AI 商品图优化。
- 官号身份：稿定商品（欢乐逛）的官方账号。
- 平台：抖音、小红书。
- 当前限制：先验证视频环节，采集和复盘飞轮暂不跑。

输出：

- 第一条样片的业务假设。
- 阶段拆分：样本结构、首条样片、模板化、后续采集复盘。

### 2. Content Strategy / 短视频结构

用途：把功能证据转成社媒视频结构。

输入：

- 功能文档。
- 商品原图与优化后 5 张图。
- 软件操作录屏。
- 用户反馈中的内容方向。

输出：

- 黄金 3 秒痛点。
- 分镜脚本。
- 口播字幕。
- 不同内容形式：操作证明型、效果展示型、批量能力证明型。

### 3. Jacky Hook / 黄金 3 秒开头闸

用途：在视频生成前，专门检查开头是否能独立抓住人。

放置位置：

- 在 Content Strategy 输出初稿后执行。
- 在 HyperFrames、video-use、Remotion 等视频生成工具之前执行。

输入：

- 初版脚本。
- 目标人群：电商商家、商品运营、美工、服装商家。
- 当前案例素材：童装平铺原图、一键优化后的 5 张上架主图、软件录屏。

输出：

- 3-5 个可直接口播的开头备选。
- 开头谜题类型判断：优先使用痛点共鸣 + WOW 效果反差。
- 开头验收结论：观众听完前 3 秒应产生“为什么这个原图问题这么明显、优化后能变成什么样”的观看动机。

验收：

- 不依赖标题和封面也能理解视频讲什么。
- 不硬喊“AI 商品图优化很强”。
- 先讲原图痛点，再承诺一键生成上架主图结果。

### 4. Jacky De-AI / 去硬广口播闸

用途：把脚本从“产品说明书口吻”改成“真人短视频口播”。

放置位置：

- Jacky Hook 通过后执行。
- 字幕、口播稿、画面提示词都先过一遍。

检查重点：

- 删除硬广口吻、空泛大词和模板句。
- 保留事实：一键优化、5 张主图、批量选商品、批量生成全套视觉、确认并发布。
- 句子变短，适合抖音/小红书竖屏字幕。
- 情绪更像商家真实痛点：原图普通、平铺没有购买欲、上架图不够撑场面。

输出：

- 可直接用于字幕的短句版。
- 可用于后续 TTS 的口播版。
- 保留信息点的改写对照。

验收：

- 不新增产品没有承诺过的能力。
- 不出现“行业革命、颠覆、最强、天花板”等高风险夸张表达。
- 每句字幕能在 9:16 画面里读完。

### 5. Jacky XHS Check / 平台风险闸

用途：发布前先检查小红书文案风险，同时作为抖音投放安全的参考。

放置位置：

- 脚本确认前检查一次。
- 最终标题、字幕、封面文案、简介、话题标签确认前再检查一次。

检查重点：

- 绝对化用语：最、第一、唯一、天花板、YYDS。
- 虚假承诺：保证、100%、必爆、必出单。
- 诱导互动和外部 CTA：关注、私信、加 V、扫码。
- 平台/竞品敏感词。
- 前后对比是否暗示不实效果。

输出：

- 风险项清单。
- 安全替代表达。
- 可发布版标题、字幕和简介。

验收：

- 不写外部导购 CTA。
- 不承诺投放效果、成交效果。
- 保留“批量生成上架主图”的功能事实，不写成“必爆主图”。

### 6. Workflow Orchestrator / 流程编排

用途：定义每一段的状态流转、输入输出和验收条件。

状态：

- `brief_ready`：推广目标与素材明确。
- `evidence_ready`：功能文档、原图、效果图、录屏齐全。
- `script_ready`：脚本和字幕确认。
- `hook_ready`：黄金 3 秒开头通过。
- `de_ai_ready`：口播和字幕去硬广完成。
- `platform_safe_ready`：标题、字幕、简介通过平台风险检查。
- `template_ready`：HTML/HyperFrames 模板可渲染。
- `screen_segments_ready`：录屏片段裁切完成。
- `video_ready`：成片合成完成。
- `qa_ready`：抽帧质检通过。
- `publish_ready`：人工确认后可进入发布工具。

关键人工确认点：

- 脚本确认后再生成视频，避免反复渲染浪费时间。
- 背景风格确认后再大规模模板化。
- 成片抽帧确认后再进入投放/发布。

### 7. HyperFrames / HTML 动效商品展示

用途：把商品图做成可复用的视频模板。

输入：

- 原图。
- 5 张优化主图。
- 样式主题：电商橱窗风、小红书笔记风。

输出：

- 大图展示片段。
- 5 张图 hover 放大片段。
- 两套背景风格 render.mp4。

验收：

- 不出现无关案例。
- 5 张图全部清楚。
- 主图表达是“上架主图”，不是“种草笔记图”。
- 字幕位置不被平台底部 UI 遮挡。

### 8. Remotion / 模板视频工程化（样片确认后接入）

用途：当当前视频形式确认后，把“一个案例一条视频”固化成代码模板。

输入：

- 商品案例 JSON：原图、5 张优化主图、商品名、行业痛点。
- 录屏片段时间点：功能入口、平台选择、主图位置、批量生成、确认并发布。
- 已通过 Jacky Hook / De-AI / XHS Check 的字幕和口播。

输出：

- 稳定的 9:16 视频模板。
- 可批量替换素材的行业案例视频。
- 可版本化管理的 React/Remotion 组件。

适用时机：

- v5/v6 样片形式被确认后。
- 需要批量生产童装、女装、鞋包、家居、食品等行业案例时。

### 9. Video-use / FFmpeg 剪辑合成

用途：裁切录屏、放大操作区域、拼接最终视频。

输入：

- 软件操作录屏。
- HyperFrames 渲染片段。
- 批量结果/发布段。

输出：

- 录屏步骤片段。
- 批量发布片段。
- 最终 9:16 MP4。

验收：

- 录屏左侧功能入口可见。
- 平台选择、主图位置、卖点/AI 标识能看清。
- 字幕够大，且位置上移。
- 画面空白不过多。

### 10. QA / 抽帧质检

用途：每轮改动后用关键帧验证。

检查点：

- 开头痛点是否明确。
- 录屏功能入口是否露出。
- 字幕是否太小或太低。
- 5 张童装主图是否都出现。
- 是否误混入女装案例。
- 批量段是否表达“批量选中商品，批量生成全套主图视觉”。
- 发布按钮是否清楚。

输出：

- `qa_report_v*.md`
- 关键帧截图目录。

### 11. Jianying Editor / 后续精剪增强

用途：当要进入更接近发布成片时，用剪映工程补充：

- 字幕样式精修。
- 转场节奏。
- 口播音频/TTS。
- 音乐和节拍点。
- 导出不同平台版本。

当前状态：

- v5 仍是验证样片，未进入剪映精剪。

### 12. Yixiaoer / 发布分发

用途：成片确认后进入多账号发布/草稿保存。

输入：

- 最终 MP4。
- 标题。
- 话题标签。
- 封面。
- 平台账号。

输出：

- 抖音/小红书草稿或发布任务。
- 发布结果记录。

当前状态：

- 仍在样片验证阶段，暂不执行发布。

### 13. 后续采集复盘飞轮

当前暂不跑，但流程预留：

- opencli 采集高赞内容。
- 结构化拆解：开头痛点、画面形式、节奏、字幕、CTA、评论反馈。
- 反哺脚本库和模板库。
- 生成下一批行业案例视频。

## 推荐端到端流程

1. `Office Hours`：锁定本轮要验证的功能、行业、账号身份。
2. `Content Strategy`：先产出脚本，不直接生成视频。
3. `jacky-hook`：打磨黄金 3 秒，确认开头有痛点、有反差、有观看理由。
4. `jacky-de-ai`：把口播和字幕改成真人表达，去掉硬广和模板腔。
5. `jacky-xhs-check`：检查标题、字幕、简介是否有平台风险。
6. 人工确认脚本。
7. `HyperFrames`：生成商品图展示段。
8. `video-use / ffmpeg`：剪录屏、放大关键操作、合成样片。
9. `QA`：抽帧检查，形成 QA report。
10. 人工反馈。
11. 回到第 2 步、第 3 步或第 7 步迭代。
12. 样片确认后进入 `Remotion` 模板化，替代手工 HTML 拼接。
13. 进入 `Jianying Editor` 精剪：字幕、音频、转场、导出。
14. 发布前再次运行 `jacky-xhs-check`。
15. 进入 `Yixiaoer` 草稿/发布。
16. 后续开启采集复盘飞轮，持续反哺脚本和模板。

## 当前组合后的主链路

```text
功能证据 + 商品素材 + 录屏
  -> Office Hours 压缩目标
  -> Content Strategy 产出脚本
  -> jacky-hook 优化开头
  -> jacky-de-ai 去硬广
  -> jacky-xhs-check 平台风险检查
  -> 人工确认脚本
  -> HyperFrames / Remotion 生成商品图动效
  -> video-use / FFmpeg 剪录屏和合成
  -> QA 抽帧质检
  -> Jianying Editor 精剪导出
  -> Yixiaoer 草稿/发布
  -> opencli 采集复盘飞轮（后续再开）
```

## 视频模板诊断环节

定位：这不是一个发布工作流节点，也不是简单的“成片是否通过”。它是模板建设工具，用来把现有视频拆开审查，反推出下一版 Remotion / HyperFrames / video-use / Jianying 模板应该怎么改。

使用时机：

- 每次生成样片后。
- 用户反馈“像广告”“不够抖音”“效果不震撼”“录屏看不清”“字幕不舒服”时。
- 准备把单条样片固化成批量模板前。

输入：

- 当前 MP4 成片。
- 关键时间点抽帧。
- 画面脚本和口播字幕。
- 目标平台：抖音、小红书。
- 目标用途：推广样片、投放素材、官号内容、批量模板。

输出：

- 不是 pass/fail，而是模板改造建议：
  - Hook 模板是否要重写。
  - 录屏模板是否要重裁、加局部放大、加光标强调。
  - 商品图展示模板是否要改成更大、更满、更有冲击。
  - 字幕模板是否要调整字号、描边、位置、安全区。
  - 节奏模板是否要减少等待、增加转场和节拍点。
  - 品牌感是否过强或过弱。
  - 哪些段落适合交给 Remotion 固化，哪些段落适合交给 Jianying 精修。

抖音推广质感检查维度：

- 前 3 秒：是否有真实痛点、视觉反差和停留理由。
- 画面占比：关键商品图和关键操作是否占画面主体，不能大面积空白。
- 结果冲击：优化后主图是否足够大、清楚、有“这能直接上架”的感觉。
- 录屏可信度：必须看得清关键操作，但不能像普通软件教学录屏。
- 字幕口播：像真人说话，不像功能说明书。
- 节奏：每 2-4 秒有信息变化，不能长时间停在静态 UI。
- 平台安全：不承诺成交、不写外部 CTA、不写极限词。

当前 v5 初步诊断结论：

- 距离“可在抖音推广”的素材还有明显差距。
- 主要问题不是单个画面错误，而是模板语言还偏“演示文稿 + 录屏拼接”，缺少抖音信息流里的冲击力、节奏和镜头设计。
- 下一版应先重构模板，再继续生成视频，不建议在 v5 上只做细枝末节修补。
